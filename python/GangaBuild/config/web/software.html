The dashboard software has three different release branches, each corresponding to a different development/testing phase. The packages are provided as (S)RPMs, binary and source tarballs. For each of the different releases there is an APT repository to ease the installation process.

All binary packages are generated using a SLC4 machine.

Stable

The stable release corresponds to the production level software.

It has been fully and is ready for overall use. This is the version of the software you'll be using if you are an end user.

Unstable

The unstable release is the necessary step before putting the software into production.

Once in unstable, a package will go under heavy testing (user end and stress testing). You should be interested in this release if you are waiting for a feature to appear in production (you can experiment it by using this release).

Nightly

The nightly release is for internal use only.

It is updated every night by going through all the build process for every single package in the dashboard software repository. It is usually not installed anywhere (used mostly for build testing).
